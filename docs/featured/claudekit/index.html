After purchasing, your GitHub username is granted access to the repos automatically. Install with these commands:

# Install ClaudeKit CLI

# Install CK Engineer in your project

ck init -g

The Battle-Tested Workflow

ClaudeKit users rely on this proven workflow to ship features with confidence:

1

/docs:init

Scans and analyzes your codebase, then generates specs: codebase structure, system architecture, code standards, product overview, and design guidelines.

2

/brainstorm

Describe what you want to build. AI asks questions to fulfill requirements, challenges you with unresolved issues, and produces a report with requirement specs.

3

/plan

Researches approaches, spawns researcher agents in parallel, and creates a comprehensive implementation plan with separate phases and validation.

4

/clear

Fresh context window before implementation to ensure best code quality output and agents stick with the plan.

5

/code:auto

Reads the plan, implements phase by phase using progressive disclosure. Write code, write tests, run tests, code review (quality & security), verification, iterate until finished, then update specs & roadmap.

This workflow is packed with 18+ years of professional development experience: Spec-driven Development, Test-driven Development, and the YAGNI - KISS - DRY principles.
