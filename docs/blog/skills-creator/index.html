What are Claude Code Skills?

Skills are reusable AI workflows that encapsulate domain knowledge, best practices, and automation logic. They allow you to create custom slash commands that provide specialized capabilities tailored to your specific development needs.

Key Concepts

Concept Description
üìã Skills Reusable AI workflows invoked via slash commands (e.g., /review-code, /generate-tests)
üìö Progressive Context Load context in layers - only what's needed, when it's needed for optimal performance
üéØ Domain Knowledge Embed specialized expertise and best practices within skills for consistent results
üîÑ Reusability Create once, use everywhere - share skills across projects and teams

Creating Your First Skill

Example: Code Review Skill

Let's create a skill that performs comprehensive code reviews:

// .claude/skills/code-review/skill.json
{
  "name": "code-review",
  "description": "Comprehensive code review with security and performance checks",
  "version": "1.0.0",
  "context": {
    "base": "context/base.md",
    "examples": "context/examples.md"
  },
  "prompts": {
    "main": "Perform a comprehensive code review focusing on security vulnerabilities, performance issues, and best practices violations."
  }
}

Context file (context/base.md):

# Code Review Guidelines

## Security Checklist
- SQL injection vulnerabilities
- XSS attack vectors
- CSRF protection
- Authentication/authorization issues
- Hardcoded secrets

## Performance Considerations
- N+1 query problems
- Unnecessary loops
- Memory leaks
- Bundle size optimization

## Best Practices
- SOLID principles
- DRY violations
- Proper error handling
- Code documentation

Progressive Context Loading

Load context progressively for optimal performance:

Layer 1: Base Context (Always Loaded)

# Essential information needed for every invocation
- Core concepts
- Basic workflow
- Critical constraints

Layer 2: Examples (Loaded on Demand)

# Code examples loaded when user needs reference
- Common patterns
- Implementation examples
- Before/after comparisons

Layer 3: Advanced (Loaded When Needed)

# Detailed information for complex scenarios
- Edge cases
- Advanced patterns
- Troubleshooting guides

Skill Configuration Options

Option Purpose
name Skill identifier used in slash command (/name)
description Brief explanation of what the skill does
context Paths to context files with progressive loading configuration
prompts Predefined prompts for common tasks within the skill
tools External tools or scripts the skill can invoke

Using Skills

Once created, invoke skills using slash commands:

# Start Claude Code
claude-code

# Invoke skill with slash command
> /code-review

# Claude will load the skill context and perform the review

# Use specific prompt variants
> /generate-tests unit

# Pass additional context
> /migrate-code react --files src/components/*.jsx

Best Practices for Skill Creation

Keep Context Focused

  • Include only essential information in base context
  • Use progressive loading for detailed examples
  • Avoid redundant information across layers

Make Skills Reusable

  • Design for multiple projects and scenarios
  • Use variables for project-specific configuration
  • Document skill prerequisites and dependencies

Optimize for Performance

  • Lazy-load expensive context only when needed
  • Cache frequently used context data
  • Minimize token usage with concise prompts

Sharing Skills

# Share via Git repository

Install Shared Skill

# Install from marketplace
npx claude-code-templates@latest --skill code-review

# Install from custom source
wget https://example.com/skills/code-review.tar.gz
tar -xzf code-review.tar.gz -C .claude/skills/

Skill Development Workflow

  1. Identify Need: Find repetitive tasks or domain expertise to encode
  2. Design Context: Organize knowledge into progressive layers
  3. Create Skill: Write skill.json and context files
  4. Test Locally: Invoke skill and refine prompts
  5. Document: Add usage examples and prerequisites
  6. Share: Publish to team or community

Skill Debugging

# Verify skill configuration
cat .claude/skills/my-skill/skill.json | jq

# Test skill without invoking
claude-code --dry-run /my-skill

# View loaded context
claude-code --debug /my-skill
